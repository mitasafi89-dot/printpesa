# 11 — Activity Feed & Chat

## 1. Live Activity feed
Left-rail stream of social-proof events (as in the screenshot): "CONGRATULATIONS @user on
withdrawal of X", win notices, "BONUS of X issued".

### 1.1 Source: mixed (real + simulated) — MVP default
- **Real events:** successful withdrawals, big wins (payout ≥ threshold), bonus issuances → inserted
  into `activity_feed` (`is_simulated=false`) and broadcast over WS `activity`.
- **Simulated events:** generated by a server job from a configurable name pool + amount ranges +
  cadence (`is_simulated=true`). Used to keep the feed lively at low traffic.

> Transparency note: simulated entries are an industry-common engagement device but must comply with
> advertising/consumer-protection rules. Admin can disable simulation entirely (real-only mode), and
> simulated/real are distinguishable in the DB for audit. **Recommended:** keep simulation modest and
> review with your legal counsel.

### 1.2 Config (admin)
- Toggle simulation on/off, name pool, min/max amounts per kind, messages/minute, real-event thresholds.

## 2. Chat
- Authenticated players post short messages (≤200 chars), streamed via WS `chat`.
- **Rate limit:** 1 msg / 2s; **profanity filter**; links/phone numbers stripped (anti-scam).
- Moderation: support can hide messages (`is_hidden`), mute or ban users.
- Stored in `chat_messages`; last N loaded on join.

## 3. Online counter
- Engine tracks live WS connections → broadcasts `online { count }` periodically (the "1,247 online").
- Admin may set a configurable floor for display; raw real count always available in admin.
